Ok, I want to get everyone reading this setup to start making webpages. I am learning Wordpress. There is a difference between WordPress.org and WordPress.com. Basically, the difference is that WP.com is for those who don't want to worry about hosting your blog and just worry about the content. WP.org is where you can download WP onto your computer and do everything else yourself, like web hosting, etc. To really know the difference, check out this link. WordPress is a beast within itself, there is so much to learn about it. If you want to start a quick blog, just go to WP.com and get started. But if you want to design a site from scratch, read on....


I'm going to give you a number of sites to use as resources as we go along. You really need to download a text editor to write code. You can download Aptana, or Notepad, gedit may already be installed on your computer. I use Sublime. You can use which ever one you want, I like Sublime, but I have also heard really good things about Aptana. Your choice! Once you've done that, you may begin.


Document types need to be setup properly, I suggest going here to learn more about that and I want you guys to start with Codecademy, this will help you get familiar with coding. Set up an account with Codecademy and start doing the first couple of lessons, trust me, you'll like it!!

November 11, 2013


Ok, so I have been going back over all the sites I'm in the process of coding and I've been retro-fitting it to be semantic and mobile-responsive. Now, it's not actually called retro-fitting in the web world, that's just a term from my construction days. It's called refactoring your code. So, that's what I did all day, refactored. And it's really quite a pain to do, so first lesson to be learned is - D.R.Y - you will see this over and over again in programming. Do Not Repeat yourself! What I really mean is when you have several pages that are linked to your home (index) page, you don't want to keep repeating yourself if those linked pages will have some if not most of the properties of your index page. I'm going to explain a little more about this tomorrow and we will get you set up with everything you need to know to start building your own website.

Anyways, on to some other things! Coding, coding is pretty radical! It's amazing the things you can do. If you know how to code and can do it well, you can do anything you want. Like create an app that helps people locate good restaurants in their area, or code a website that helps you keep track of the important things you need to do on a particular day. These ideas are obviously already taken, but you can come up with any new idea and you can always improve on the ideas that are already out there.


So, I've been learning some pretty neat stuff. HTML and CSS laid the ground work for me. You must learn these two things first. HTML is the bones or skeleton of your website. It's a document where you put all of your text and headers, paragraphs and sidebars, and much more. It's essential to creating a website. CSS lets you decorate that website. You can make your header look like block lettering or make your navigation list change color when the mouse hovers over it. I have just recently begun JavaScript and jQuery, and these two things let the user of your website interact with it. It can animate something or make an image explode. It's pretty cool!
